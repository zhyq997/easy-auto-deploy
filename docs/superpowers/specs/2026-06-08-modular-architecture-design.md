# 模块化架构设计

## 目标

将当前单文件 CLI 拆分为职责清晰、可独立测试的模块，同时保持现有部署、回滚和备份保留行为不变。

## 模块边界

- `src/config.mjs`
  - 加载业务项目中的 `deploy.config.js`。
  - 校验构建命令、构建目录、服务器连接信息和部署目录。
  - 将 `privateKey` 路径兼容转换为 `privateKeyPath`。
- `src/archive.mjs`
  - 执行构建命令。
  - 压缩构建产物。
  - 清理本地临时压缩包。
- `src/ssh-client.mjs`
  - 封装 `node-ssh`。
  - 连接服务器、上传文件、执行并检查远程命令。
  - 通过 SFTP 获取和删除备份文件。
- `src/deploy-service.mjs`
  - 编排构建、压缩、连接、上传、部署和备份清理。
  - 提供版本列表和回滚能力。
  - 通过构造参数接收归档和远程适配器，便于测试。
- `src/cli.mjs`
  - 展示交互菜单和确认提示。
  - 将用户选择委托给部署服务。
  - 统一展示成功、取消和错误信息。
- `index.mjs`
  - 创建真实依赖并启动 CLI。
  - 被作为模块导入时不自动执行。

## 数据流

发布流程：

1. CLI 调用 `deployService.deploy()`。
2. 服务生成版本名并调用归档适配器构建、压缩。
3. 服务连接远程服务器并上传 ZIP 到 `backups`。
4. 远程适配器清理部署目录并解压目标版本，非零退出码视为失败。
5. 服务获取按修改时间倒序的 ZIP 列表并删除第 6 个及之后的文件。
6. 无论成功失败，都清理本地 ZIP 并释放 SSH 连接。

回滚流程：

1. CLI 获取远程备份列表并让用户选择。
2. 服务连接服务器并部署选中的 ZIP。
3. 回滚不触发备份清理。

## 错误处理

- 底层模块抛出 `Error`，不调用 `process.exit()`。
- CLI 捕获可展示错误；入口设置 `process.exitCode = 1`。
- 远程命令以退出码为准，`stderr` 仅作为错误详情或警告信息。
- 配置缺失或字段无效时，在连接服务器前失败。

## 测试

- 配置：有效配置、缺少字段、危险部署目录、私钥路径兼容。
- 部署服务：调用顺序、失败传播、资源清理、备份裁剪、回滚不裁剪。
- SSH 适配器：SFTP 列表排序和过滤、远程非零退出码。
- CLI：发布、回滚、取消和错误返回码。
- 现有备份保留纯函数测试继续保留。

## 非目标

- 本轮不实现软链接原子发布、多环境、健康检查、通知或部署锁。
- 不改变现有配置文件名称和交互菜单入口。
